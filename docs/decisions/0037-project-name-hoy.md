# 0037. プロジェクト名は hoy

- **Status**: Accepted
- **Related**: —

## Context

これまで仮称として `nextgit` を使ってきたが、正式なプロジェクト名を決める必要があった。命名の軸として:

- 短くタイプしやすい(git に倣う)
- 意味は薄め(自虐ネーミングの git に倣う)
- 検索性・ドメイン取得可能性
- 日本語話者・英語話者ともに発音しやすい

## Decision

プロジェクト名は **`hoy`** とする。

## Rationale

QWERTY キーボード配列で **`git` の各文字の右隣のキー**:

- `g` の右隣 → `h`
- `i` の右隣 → `o`
- `t` の右隣 → `y`

つまり `hoy` は物理的に「git の隣」=「次の git」=「ne**xt** git」を体現している。タイプ時も `git` を打つときから指を1つ右にずらすだけで `hoy` になる。

副次的特徴:

- 3文字・1音節
- 日本語の「ほい」(何かを手渡すときの掛け声)とも音が一致 — Intent → Task をエージェントに渡すモデルと文化的に共鳴
- "ahoy"(呼びかけ)や Spanish "hoy"(today)との連想もある
- 既存大手プロダクトとの衝突が少ない(要最終確認)

## Consequences

- ドキュメント・ADR・コード・パッケージ名で `hoy` を使う
- 既存ドキュメントの `nextgit` 表記は順次 `hoy` に置き換える(本ADRと同一PR内で対応)
- 作業ディレクトリ名 `/Users/hiragram/Development/nextgit/` のリネームはユーザー判断
- ドメイン取得・GitHub org 名等の確保は別途
- バイナリ名・CLI コマンド名も `hoy` を採用(例: `hoy intent create ...`)
