# 0003. Task起票主体は人間/エージェント両方を許容

- **Status**: Accepted
- **Related**: open-questions #1

## Context

Intent から Task への分解を誰が行うか。人間専用、エージェント専用、両方許容、の選択肢があった。

## Decision

Intent を Task に分解するのは人間でもエージェントでもよい。CLI から人間が、MCP からエージェントが、それぞれ同じ操作を行える。

## Rationale

- 「エージェントしか分解できない」「人間しか起票できない」のどちらに倒しても現実と合わない
- データモデル上、両者を等価に扱うことで運用の自由度が増す

## Consequences

- 起票主体を区別するためのメタデータが必要(→ ADR 0006)
- CLI と MCP で操作が等価である必要がある(→ ADR 0007)
- 「エージェントが分解した Task の妥当性を人間が承認するか」は別途議論が必要(現時点では未定)
