# 0033. hoy本体はLLM推論コストを負担しない

- **Status**: Accepted
- **Related**: open-questions #7

## Context

原版 open-questions #7 では「意味レベル差分解析・リアルタイム調停・検証経路の自動分類すべて LLM 推論が要る」とされ、コスト・レイテンシが懸念だった。これまでの議論でその前提が大きく変わったため、明示的に確認しておく必要があった。

## Decision

hoy 本体は **LLM 推論コストを内部で負担しない**設計とする。LLM が必要な処理(意味レベルの判断、Intent からの自動 Task 分解、ドリフト検出等)はすべて外部 agent / hook 側の責務。

## Rationale

これまでの ADR で以下が確定している:

- ADR 0002: hoy はエージェント機能を持たない
- ADR 0013: Git 内部利用、差分はテキストレベル
- ADR 0017: コンフリクト解決は Git rebase ベース、LLM 不要
- ADR 0021: ドリフト検出はメタデータ提供のみ、判定は外部

これらは独立に決まったが、結果として「hoy 本体は決定的・非LLM」という性質が浮かび上がっている。これを明文化することで、将来「daemon にLLMを入れたい」という提案が出たときの判断軸になる。

## Consequences

- daemon は決定的に動作する。同じ入力に対して同じ出力を返す
- daemon の動作コストは LLM API の従量課金ではなく、CPU・メモリ・ディスクのみ
- LLM 関連のコストカーブ(将来安くなる/高くなる)は本プロダクトの直接の経済性に影響しない
- 高度な「意味理解」を必要とする機能は外部 hook / agent としてのみ実装される
- 検証経路の `automated` check が外部 LLM API を叩くことは可能だが、それは check の実装側の責任

## 性能要件について

具体的な性能目標値(API レイテンシ、メモリ使用量、スループット等)は本ADRでは定めない。MVP 実装段階で実測値を踏まえて決める。
