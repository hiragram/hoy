# 0002. プラットフォーム自体はエージェント機能を持たない

- **Status**: Accepted
- **Related**: open-questions #1

## Context

nextgit が提供するのはデータモデル・プロトコル・CLI・MCP インタフェースであり、LLM 推論やエージェント実行をプラットフォームに内包するかが論点だった。

## Decision

nextgit のエコシステム自体はエージェント機能を持たない。エージェントは外部(Claude Code, Cursor, 独自実装など)から MCP 経由で接続する。

## Rationale

- Git がエディタを内包しなかったように、エージェントを内包しないことでエコシステムの広がりが大きくなる
- どのエージェント技術(現在/未来)とも組める
- エージェントが一切いない運用(全部人間)も成立する
- エージェント技術の進化と無関係に、データモデルだけは生き残れる

## Consequences

- エージェント連携はすべて MCP 経由になり、プロトコル設計が一級市民
- プラットフォームは LLM 推論コストを負担しない
- 「推奨エージェント」のような縛りは持たない(参考実装は提供しうる)
