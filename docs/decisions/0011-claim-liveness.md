# 0011. claimの生存管理はハートビート+強制release

- **Status**: Accepted
- **Related**: open-questions #2

## Context

ADR 0009 で Intent に排他 claim を導入したが、claim したエージェントがクラッシュ・無応答になった場合に claim が永遠に残るリスクがある。

## Decision

claim の生存管理は以下の組み合わせで行う:

- **ハートビート方式**: claim 中のエージェントは定期的に生存報告を送る。一定時間応答が途絶えたら自動 release。
- **強制 release**: 人間 or 別のエージェントが「死んでいる」と判断した場合、強制的に claim を奪える。

## Rationale

- ハートビート単体では、ネットワーク不調等で誤って release されるリスクがある → 強制 release を補完として併用
- 明示的な TTL のみだと、長時間タスクで逐一更新が必要になり煩雑
- 自動回復 + 人間の最終介入の二段構えが現実的

## Consequences

- claim には `last_heartbeat_at` のような時刻情報が必要
- ハートビート間隔・タイムアウト値はデフォルトを定めつつ、設定可能にする
- 強制 release は監査ログに残す(誰が・いつ・なぜ奪ったか)
- ハートビート未対応のクライアント(雑なスクリプト等)向けの代替手段は別途検討
