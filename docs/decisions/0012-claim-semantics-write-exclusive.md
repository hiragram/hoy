# 0012. claimの意味論は「書き込み排他・読み取り自由」

- **Status**: Accepted
- **Related**: open-questions #2

## Context

ADR 0009 で導入した排他 claim の厳密な意味論(claim 中、他のエージェント/人間に何が許されるか)を定める必要があった。

選択肢:
- (c1) 完全排他: 読み取りのみ可
- (c2) 書き込みのみ排他: 閲覧・コメント・観測は可
- (c3) 助言モード: 提案 Task を作れる(claim 者の承認で正規化)

## Decision

**書き込みのみ排他**。claim 中、claim 者以外は以下が可能:

- Intent 内容の閲覧
- 配下 Task の閲覧
- コメント・観測

許されないこと:

- Task の作成・更新・削除
- Intent 自体の更新
- 関連する実装の確定

## Rationale

- 完全排他は厳しすぎ、観測すら止めるのは並列運用の利点を損なう
- 助言モードは複雑で、最初に入れる価値が薄い(将来的に追加する余地は残す)
- 「書き込み」「読み取り」のシンプルな線引きが API 設計上もクリーン

## Consequences

- API レベルで「書き込み系」操作の権限チェックが必要
- コメント機能は claim とは独立の権限モデルを持つ
- 将来的に助言モード(c3)を追加する場合は、本 ADR の Superseded もしくは追補で扱う
